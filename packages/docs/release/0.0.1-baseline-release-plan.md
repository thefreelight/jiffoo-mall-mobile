# Baseline Release Plan: v0.0.1

## Purpose

`v0.0.1` is the first documented foundation baseline release.

It is not trying to claim that the entire foundation is mature.

It is trying to establish a clean, reproducible adoption point for early consumers.

## Why `0.0.1`

We are intentionally using `0.0.1` because:

- the repository is now coherent and runnable
- release and upgrade rules are documented
- compatibility lab smoke checks exist
- but several runtime capabilities are still experimental rather than broadly stable

This gives downstream adopters a known baseline without over-promising maturity.

## Included Baseline Scope

- `design-tokens`
- `design-system`
- `navigation`
- `permissions`
- `storage`

## Explicitly Held Back

- `networking`
- `observability`
- `debug-tools`
- host shells
- demo gallery UI

These remain useful in-source, but should not be presented as baseline consumer-facing release units yet.

## Release Readiness Checklist

- [x] Public/private boundary documented
- [x] Android public host runnable
- [x] iOS public host runnable
- [x] Demo galleries available on both platforms
- [x] Stability tiers documented
- [x] Distribution blueprint documented
- [x] Compatibility lab implemented and passing locally
- [x] Release notes written for `v0.0.1`
- [x] Migration guide written for `v0.0.1`
- [x] Repository tag `v0.0.1` created on the release branch

## Downstream Consumer Message

Consumers should read this baseline as:

- a safe starting point for early adoption
- a source-compatible foundation with documented upgrade intent
- not yet a promise that every capability is fully stable

## Follow-Up After `v0.0.1`

- stabilize `observability`
- stabilize `debug-tools`
- decide whether `networking` can move from experimental toward stable
- publish actual Android artifacts and consumer-facing Swift Package guidance
