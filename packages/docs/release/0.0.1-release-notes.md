# Release Notes

## Release

- Version: `v0.0.1`
- Date: `2026-04-23`
- Release type: `patch`

## Summary

`v0.0.1` is the first public baseline for `mobile-foundation`. It establishes a runnable Android/iOS foundation host, documented public boundaries, and the first contract-driven storefront reference flow that downstream apps can build on.

## Included Release Units

- Android:
  - foundation host app
  - `core/runtime`
  - `core/navigation`
  - `core/designsystem`
  - `core/networking`
  - `core/storage`
  - `core/permissions`
  - `core/observability`
  - `core/debug`
- iOS:
  - foundation host app
  - `AppShell`
  - `Navigation`
  - `DesignSystem`
  - `Networking`
  - `Storage`
  - `Permissions`
  - `Observability`
  - `DebugTools`
- Source-of-truth assets:
  - design tokens
  - docs / ADR / spec / PRD / execution chain
  - release blueprint and compatibility checks

## Highlights

- Added:
  - native Android and iOS foundation hosts
  - public capability modules across both platforms
  - storefront contract probes for store, theme, catalog, cart, and checkout entry
  - compatibility lab and release-readiness checks
- Changed:
  - repositioned the repository into a reusable foundation/platform instead of a product app repo
  - clarified the public/private boundary for downstream product apps
- Fixed:
  - aligned Android and iOS reference hosts around the same core storefront contract

## Consumer Impact

- Safe to upgrade without code changes:
  - yes, for first-time adopters starting from this baseline
- Requires follow-up changes:
  - downstream apps still need their own auth lifecycle, branding, and product-specific flows
- Experimental modules affected:
  - storefront reference flows remain examples of the public contract, not a finished consumer-facing product shell

## Migration Notes

See [0.0.1-migration-guide.md](/Users/jordan/Projects/jiffoo-mall-mobile/packages/docs/release/0.0.1-migration-guide.md).

## Known Limitations

- live authenticated flows such as cart and checkout require a manually supplied token in the public host
- payment session orchestration and full order management are not part of this baseline release
- downstream consumers should still treat the reference storefront flow as guidance, not as a ready-made production app
