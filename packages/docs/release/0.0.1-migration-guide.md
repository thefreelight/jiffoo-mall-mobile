# Migration Guide

## Version Transition

- From: `unreleased foundation work`
- To: `v0.0.1`

## Who Should Read This

Teams starting a private mobile app on top of `mobile-foundation`, or any maintainer who plans to mirror this baseline into a downstream product repo such as `Bokmoo_app`.

## Breaking Changes

- None. This is the first tagged baseline.

## Deprecations

- None.

## Required Changes

1. Consume the tagged baseline instead of an arbitrary branch snapshot.
2. Decide whether your downstream app will vendor the foundation source or consume future artifact/package releases.
3. Keep product-specific auth, branding, and business flows in the downstream repo rather than adding them back into the public foundation.

## Validation Steps

- Re-run compatibility lab
- Rebuild Android host
- Rebuild iOS hosts
- Rebuild downstream consumer integration if applicable

## Notes

- The public storefront flow is intentionally contract-driven and conservative.
- `builtin-default` is the only baseline path that should be treated as the official starting point for downstream adoption at this stage.
